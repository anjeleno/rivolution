# Specs

Technical specs for patches applied to this fork. Each spec covers one
atomic, mergeable unit of work: exact files, functions, data formats, and
behavior decisions, written for someone reading the source — not project
narrative.

Scope rule: keep this directory limited to technical design content only.
No internal strategy discussion, no commentary about people or
organizations, no credentials or personal contact information. This repo
may be made public again in the future, so write every file here as if it
already is.

Project history, planning conversations, and anything not meant to be
public live outside the repo entirely, not here.

Naming: numbered sequentially in the order patches were specced, mirroring
`git format-patch` output — `0001-feature-name.md`, `0002-next-feature.md`.
